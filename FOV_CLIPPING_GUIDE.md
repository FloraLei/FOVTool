# FOV遮挡切割功能指南

## 问题背景
在3D视图中，后置传感器的FOV显示时没有被挂车遮挡，导致可视化不符合实际情况。

## 解决方案
新增了灵活的FOV切割功能，支持两种模式：**自动切割** 和 **手动调整**。

## 功能说明

### 1. 自动遮挡切割（推荐）
- **参数**: `被车体遮挡时切割FOV` 复选框
- **默认状态**: ✓ 开启
- **工作原理**: 系统自动计算FOV与车体的交集
  - 对于后置传感器：FOV被切割到车体后端（或挂车后端）
  - 对于前置传感器：FOV被切割到车体前端
  - 不影响侧方传感器

### 2. 手动微调距离
- **参数**: `手动切割距离` (0-200 m)
- **默认值**: 0 m（表示自动计算）
- **使用场景**:
  - 需要看到部分被遮挡的FOV（设置较小的值）
  - 强制完全剪裁FOV（设置较大的值）

## 使用流程

### 步骤1：在属性编辑面板中调整
1. 在左侧"传感器列表"中选择后置传感器
2. 在右侧属性面板找到新增的选项：
   - `被车体遮挡时切割FOV` ☑
   - `手动切割距离: 0 m`

### 步骤2：根据需要调整
```
默认情况（自动切割）:
┌─────────┐
│ ✓ 被车体遮挡时切割FOV   ← 勾选开启自动切割
│ 手动切割距离: 0 m       ← 设为0表示自动计算

调整显示范围:
┌─────────┐
│ ✓ 被车体遮挡时切割FOV
│ 手动切割距离: 50 m      ← 调整为50m，显示到距车体50m处

禁用切割（显示完整FOV）:
┌─────────┐
│ ☐ 被车体遮挡时切割FOV   ← 取消勾选
│ 手动切割距离: 0 m
```

### 步骤3：3D视图实时更新
- 修改完毕后，3D视图会实时重新渲染
- 后置传感器的FOV会被正确切割

## 技术细节

### 切割算法
```
切割平面位置 = min(
    车体后端: -vehicle.length,
    挂车后端: -vehicle.length - trailer.length
)

保留条件: FOV点的Y坐标 > 切割平面Y坐标
```

### 参数自动验证
- `clip_by_vehicle = True`: 启用遮挡切割
- `clip_range = 0.0`: 使用自动计算
- `clip_range > 0`: 使用手动指定的距离

## 常见场景

### 场景1：后置摄像头（完整遮挡）
```
配置:
  - 被车体遮挡时切割FOV: ✓ 开启
  - 手动切割距离: 0 m（自动）

结果: FOV被完全切割到车体后端
```

### 场景2：侧面摄像头（部分可见）
```
配置:
  - 被车体遮挡时切割FOV: ✓ 开启
  - 手动切割距离: 0 m

结果: FOV自动与车体边界对齐
```

### 场景3：禁用切割（查看原始FOV）
```
配置:
  - 被车体遮挡时切割FOV: ☐ 关闭

结果: 显示完整FOV，忽略车体遮挡
```

## 影响范围

### 仅影响3D视图
- ✅ Canvas3D渲染
- ❌ 不影响2D俯视图
- ❌ 不影响PNG导出（导出已有独立的遮挡处理）

### 实时有效
- 修改参数后立即生效
- 无需保存或重新启动

## 性能影响
- **自动切割**: 轻微性能开销（每帧计算一次）
- **建议**: 对于复杂场景（10+传感器），可关闭某些传感器的切割

## 数据持久化
- FOV切割参数随传感器配置保存到JSON
- 加载配置时自动恢复设置

## 故障排除

### Q: 切割后FOV完全消失了
**A**: 检查以下几点：
1. 确认`手动切割距离`不是太大
2. 尝试将其设为0（自动计算）
3. 检查传感器的Y坐标位置

### Q: 看不到效果变化
**A**: 
1. 确保传感器是**后置**（Y值为负）
2. 尝试旋转3D视图以观察变化
3. 确认`被车体遮挡时切割FOV`已勾选

### Q: 如何恢复原始FOV显示？
**A**: 
1. 取消勾选`被车体遮挡时切割FOV`
2. 点击OK或等待自动保存
